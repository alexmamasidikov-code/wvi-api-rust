## 4. Phase B — WVI Formula QA ✅

### Deliverables
- `docs/qa/wvi_validator.py` — Python reference port of `calculator.rs`.
- `docs/qa/test-vectors/wvi_vectors.json` — **33 golden vectors** covering every tier + edge cases + hard caps + emotion coverage + boundary conditions + realistic daily scenarios.
- `docs/qa/regenerate_vectors.py` — vector regenerator.
- `src/wvi/calculator.rs` `#[cfg(test)] mod tests` — invariant-based parametric runner + fail-safe tests.

### Invariants verified for all 33 vectors
- `score ∈ [0, 100]` & finite
- Per-metric scores `∈ [0, 100]` & finite
- Category median order: superb > excellent > good > moderate > attention > critical > dangerous

### Fail-safe tests (all pass)
| Test | Input | Verified |
|---|---|---|
| `zero_inputs_dont_panic_or_yield_nan` | all 14 metrics = 0 | finite, clamped |
| `negative_inputs_clamp_not_panic` | HR=-50, HRV=-10, stress=-100, spo2=-5 | finite, [0, 100] |
| `absurdly_large_inputs_clamp` | HR=500, HRV=1000, steps=1M | finite, [0, 100] |
| `empty_emotion_name_is_neutral` | emotion_name="" | multiplier = 1.0 |

### 🔧 Real bug fixed
`calculator.rs:232` — `sleep_score` / `emotion_score` now clamped to [0, 100] before aggregation. Previously an input like `sleep_score=200` passed verbatim into the geometric-mean input, producing metric_scores out of [0, 100] range and breaking downstream invariants. Caught by `all_vectors_invariants_hold` on the `edge_absurd_high` vector.

## 6. Phase D — Computed Metrics QA ✅

16 new tests across 6 derived indicators:

| Indicator | Tests | Coverage |
|---|---|---|
| **Blood Pressure** | 4 | normal / elevated / clamp boundaries / zero |
| **VO2 Max** | 3 | young fit / older moderate / 15-75 clamp |
| **Coherence** | 2 | monotonic with HRV / 0-100 clamp |
| **Sleep Score** | 3 | ideal / poor / zero inputs |
| **Bio Age** | 3 | healthy < chrono, unhealthy > chrono, zero safe |
| **Training Load** | 4 | 200 TRIMP cap / 120 min cap / negative / **zero hr_max** |

### 🔧 Real bug fixed
`computed.rs:compute_training_load` — guard against `hr_max=0`. Previously `avg_hr / 0` produced `Inf`, which `clamp(0.0, 1.0)` silently converted to 1.0 (masking the bug), but the intent was undefined behavior. Now defaults to `hr_max_safe = 180.0` when `hr_max <= 0`, and clamps negative `active_minutes`/`avg_hr` to 0. Caught by `training_load_zero_hr_max_doesnt_panic`.
